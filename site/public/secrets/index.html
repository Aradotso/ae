Secrets live in Infisical.

One source of truth for every runtime credential the team uses — grouped into folders by service inside the Ara-passwords project. Agents connected to the Ara MCP fetch them via infisical_* tools. Humans use 1Password.

How services read them

Services wrap their start command with infisical run. Railway holds just the machine-identity bootstrap (INFISICAL_CLIENT_ID + INFISICAL_CLIENT_SECRET). Everything else is fetched on container start. 

# In mcps/Dockerfile (and analogous for every service):
CMD ["infisical", "run", \
     "--projectId=6d518288-7854-49d2-aa42-8ffd285dafa1", \
     "--env=prod", "--path=/mcp", "--recursive", \
     "--", "bun", "run", "src/index.ts"]

How agents read them

The Ara MCP server exposes infisical_* tools. Pass env + path to scope. 

{
  "tool": "infisical_get_secret",
  "args": {
    "env": "prod",
    "path": "/shared",
    "key": "OPENAI_API_KEY"
  }
}

The rules

1. Never commit secrets.

.env and .env.local are gitignored. Local dev: infisical secrets -o dotenv --env=dev --path=/<service> > .env.local. If a key leaks to git, rotate in Infisical first, then worry about history.

2. Never roll your own vault.

Infisical is the store. 1Password is for humans. Don't build anything in between — no custom service, no shared sqlite, no "just for dev" helper.

3. Don't put new secrets in Railway.

Railway is a runtime host, not a vault. Add secrets to Infisical and let infisical run load them. The only Railway vars that stay are Railway-auto ones (RAILWAY_*, PORT) and the Infisical machine-identity bootstrap.

4. Agents don't ask for keys.

If an agent needs a credential, it calls infisical_get_secret. If the key isn't there, the wiring is broken — add it to Infisical, don't paste it in chat.

5. Rotation is an Infisical edit.

Update the value in Infisical, redeploy the service, done. If the key is shared across services, put it in /shared/ once — don't copy into each service's folder.

6. Human secrets stay in 1Password.

Personal Google tokens, SSH keys, and anything tied to a human identity don't go in Infisical. Infisical is for what services need to run.
